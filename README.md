# Anchor Group — Monthly Metrics

Zillow Preferred performance reporting + monthly digest pipeline for The Anchor
Group.

- Researches the **current** Zillow Preferred KPIs each month (the program
  changes; the metric set is not hard-coded).
- Ingests admin-uploaded CSV/JSON exports (or pulls live from Follow Up Boss).
- Persists history in SQLite for 30/60/90-day rolling trends and team benchmarks.
- Renders per-agent HTML emails (Jinja2) and a Reveal.js team meeting deck.
- Ships a mobile-first Flask + HTMX + Tailwind admin dashboard with a
  human-in-the-loop **draft → approve → send** workflow.
- Runs on a HEARTBEAT schedule (cron / systemd timer) at 09:00 on the 1st
  of each month.

## Quick start

```bash
# 1. Install
pip install -r requirements.txt

# 2. Configure secrets
export ANTHROPIC_API_KEY=sk-ant-…    # for monthly KPI research
export ADMIN_PASSWORD=choose-strong  # dashboard login (default: 'anchor')
export SMTP_USER=reports@anchorgroup.com
export SMTP_PASSWORD=…
# Optional, only if pulling from FUB live:
export FUB_API_KEY=…

# 3. Refresh KPIs from the live Zillow program
python main.py --mode research

# 4. Upload a month of data (or pull from FUB)
python main.py --mode upload tests/fixtures/april_2026_sample.csv

# 5. Queue draft emails
python main.py --mode draft

# 6. Review + approve via the dashboard
python main.py --mode dashboard
# → http://127.0.0.1:5050
```

## Modes

| Mode        | What it does                                                           |
|-------------|------------------------------------------------------------------------|
| `research`  | Asks Claude (with web search) for the current Zillow Preferred KPIs and writes `config/thresholds.json`. |
| `upload`    | Parses a CSV/JSON file into `data/metrics.db`. Required.               |
| `review`    | Renders all agent emails + team deck to `output/review/` for local preview. |
| `draft`     | Like `review`, but persists drafts to the approval queue (no files written). |
| `dashboard` | Starts the Flask admin UI on `:5050`.                                  |
| `send`      | SMTP-sends every draft currently in `status='approved'`.               |

Use `--source fub` to override the default and pull live from Follow Up Boss.
Use `--period YYYY-MM` to load a specific historical month from SQLite.
Use `--mock` for development without any external API key.
Use `--dry-run` with `send` to print recipients without delivering.

## CSV format

```
agent_id,name,email,period,<metric_key_1>,<metric_key_2>,…
mock-001,Alex Rivera,alex@example.com,April 2026,180,0.62,0.91,0.71,0.78
```

Metric columns must match the keys in `config/thresholds.json`. If Zillow
changes the program, run `python main.py --mode research` and the dashboard
will tell you which columns the next upload needs.

## Architecture

```
config/
  settings.py        Brand palette, SMTP creds, agent roster, paths.
  thresholds.json    Current KPIs + targets (auto-generated by --mode research).

src/
  threshold_researcher.py  Claude-API-driven KPI discovery.
  fub_client.py            Optional live Follow Up Boss pull.
  csv_ingest.py            CSV/JSON admin ingest.
  storage.py               SQLite persistence + draft approval queue.
  metrics.py               Scoring engine (dynamic registry, lower-is-better support).
  gauges.py                Inline SVG arc gauges.
  email_builder.py         Per-agent HTML email rendering.
  deck_builder.py          Team meeting Reveal.js deck.
  review_mode.py           Static HTML preview output.
  dashboard.py             Flask + HTMX + Tailwind admin UI.

templates/
  email.html.j2            Per-agent monthly digest.
  deck.html.j2             Team meeting deck.
  admin/                   Dashboard templates (Tailwind via CDN).

scripts/
  heartbeat.sh             Cron-friendly monthly trigger.

data/                      SQLite history (gitignored).
logs/                      HEARTBEAT logs (gitignored).
tests/                     pytest suite — round-trip ingest, scoring, privacy.
```

## Privacy

Per-agent emails are rendered with that agent's record only — the Jinja
context contains a single `agent` object, not the full team. The dashboard is
admin-only (password-gated). Tests in `tests/test_privacy.py` assert no
cross-agent data leaks into rendered emails.
