## The one thing that doesn't auto-pull

**Zillow Preferred Performance Report data.**

What I confirmed against the live FUB account:

- `GET /v1/users` → **200**, 29 users
- `GET /v1/identity` → **200**, account info
- `GET /v1/me` → **200**, your user record
- `GET /v1/reporting/zillow-preferred?...` → **404**, "Collection name '' is not valid"
- `GET /v1/reporting/agent?...` → **404**, same error
- `GET /v1/agents` → **404**, "use 'people'"

And [FUB's own help center](https://help.followupboss.com/hc/en-us/articles/38151364189079-Zillow-Preferred-agent-performance-report) describes the ZP Performance Report as something you "view by logging in" — there's no API mention.

So the four scoring metrics (`pCVR`, `pickup_rate`, `csat`, `zhl_transfers`) cannot be auto-pulled from FUB today. Your options:

1. **Stay on CSV upload.** The dashboard already supports it (`d8c7025 feat: monthly digest system — dynamic KPIs, CSV ingest`). Each month, you export the ZP CSV from FUB UI and upload it via `https://anchor.joelycannoli.com`. The cron's failure email becomes your reminder to do that. **Simplest, most reliable.**
2. **Email-to-ingest.** Schedule FUB to email you the report monthly, and have a small mailbox watcher drop the CSV into `/opt/Monthly-Metrics/data/incoming/`. We can build that on top of the existing CSV ingest.
3. **Compute metrics ourselves from raw FUB data** (people, calls, deals via the public API). Doable but the calculations won't match Zillow's numbers exactly — pCVR in particular is Zillow's own model. Probably not worth it.
4. **Ask FUB support** if there's a private/partner endpoint for the ZP report. Worth a ticket.

My recommendation: **(1) for now**, with **(4) as a parallel ask**. The cron + alert + CSV upload workflow is robust and you'll get data on Day 1 of every month either way.
