[tool.ruff.format]

[tool.mypy]
python_version = "3.11"
strict = false

[tool.coverage.run]
branch = true
source = ["src", "config", "main"]
omit = ["*/tests/*", "*/__init__.py"]

[tool.coverage.report]

skip_covered = false

## Task 9: Add `.github/workflows/ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

CI must pass on every PR to `main`. P0 baseline is non-blocking (lint warnings allowed) — full strict mode lands incrementally in P1/P3.

- [ ] **Step 1: Create the workflow file with the full content below**

```yaml
name: CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

jobs:
  test:
    name: Lint + Type-check + Test (Python ${{ matrix.python-version }})
    runs-on: ubuntu-latest
    strategy:
      fail-fast: false
      matrix:
        python-version: ["3.11", "3.12"]

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Python ${{ matrix.python-version }}
        uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
          cache: pip

      - name: Install dependencies
        run: |
          python -m pip install --upgrade pip
          pip install -r requirements.txt
          pip install ruff mypy pytest pytest-cov

      - name: Ruff lint
        run: ruff check src config main.py

      - name: Ruff format check
        run: ruff format --check src config main.py

      - name: Mypy
        run: mypy src/ || true       # Non-blocking until P3.

      - name: Pytest with coverage
        run: pytest -q --cov --cov-report=term --cov-report=xml

      - name: Upload coverage XML (3.11 only)
        if: matrix.python-version == '3.11'
        uses: actions/upload-artifact@v4
        with:
          name: coverage-xml
          path: coverage.xml
          retention-days: 14
```

- [ ] **Step 2: Validate YAML locally**

Run:
```bash
python -c "import yaml; yaml.safe_load(open('.github/workflows/ci.yml'))" && echo OK
```
Expected: `OK`. If `yaml` is not installed: `pip install pyyaml` first.

## Rollback runbook

If something goes wrong before the PR merges, you can revert all branch surgery:

1. Re-create deleted branches from archive tags:
   ```bash
   git push origin archive/claude/notebooklm-mcp-access-js94b:refs/heads/claude/notebooklm-mcp-access-js94b
   git push origin archive/claude/zillow-digest-system-OGMIF:refs/heads/claude/zillow-digest-system-OGMIF
   git push origin archive/claude/analyze-test-coverage-pKsDU:refs/heads/claude/analyze-test-coverage-pKsDU
   ```
2. Reset the GitHub default branch back to `claude/notebooklm-mcp-access-js94b` in the UI.
3. Delete `main` if it has no merged PRs:
   ```bash
   git push origin --delete main
   ```
4. Worst case (everything corrupted) — restore from the bundle:
   ```bash
   cd /tmp && git clone data/backups/pre-reset-2026-05-07.bundle Monthly-Metrics-restored
   ```

After the PR merges, rollback is `git revert <merge-sha>` on `main` for the file changes; branch surgery is permanent (intentionally — that was the whole point).
